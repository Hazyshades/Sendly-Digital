Fixed Semaphore deployment in `deploySemaphore.mjs`.
What was done:
- added auto-deploy for the `PoseidonT3` library if no address is set;
- implemented bytecode linking for Semaphore with the library (otherwise there was an invalid BytesLike error);
- added a configurable RPC timeout and a fixed network via `.env`;
- deployment now completes fully (Verifier -> PoseidonT3 -> Semaphore).

Why:
- without library linking the bytecode is invalid and the contract cannot be deployed;
- timeout and a static network reduce failures with unstable RPC;
- as a result we get correct addresses for all contracts after running the script.
